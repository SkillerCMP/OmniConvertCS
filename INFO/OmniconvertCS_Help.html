1. Overview

Omniconvert C# is a reimplementation of the classic Omniconvert tool, written in C# / WinForms. It converts PlayStation 2 cheat codes between various devices and formats:

  • Action Replay V1 / V2
  • Action Replay MAX (ARMAX) – encrypted and RAW
  • GameShark (various versions)
  • CodeBreaker (CBC, encrypted & RAW)
  • XP / Xploder
  • Standard RAW
  • PCSX2 PNACH (RAW patch format)
  • Swap Magic Coder

Version 1.01 (C# modular build) adds:

  • INLINE input mode – per-cheat , CRYPT_XXXX tags in one paste.
  • PNACH (RAW) input & output with CRC support and a dedicated CRC dialog.
  • Per-cheat error reporting – failed cheats emit clear error comment blocks instead of silently disappearing.
  • UI tweaks such as AR2 key display and PNACH CRC “Add CRC” checkbox integration.

2. Main window tour

2.1 Input panel (left)

  • Input text box – paste or type your source codes here.
  • Clear – clears the input box.
  • Game ID (Input) – numeric ID used by some devices (e.g., ARMAX, organizers).
  • Input format label – shows the current input selection, for example: Input: Standard(RAW) or Input: INLINE (from ', CRYPT_XXXX' headers).

2.2 Output panel (right)

  • Output text box – converted output appears here after pressing Convert.
  • Clear – clears the output box.
  • Game ID (Output) – tracks the game ID of the converted codes (used when saving certain formats).
  • Game Name – friendly game name for PNACH and ARMAX data.
  • Output format label – shows the current output selection, such as: Output: Pnach(RAW) or Output: Codebreaker(RAW).
  • Add CRC (checkbox – visible when Output is PNACH(RAW)) – toggles whether PNACH output includes a CRC header.

2.3 Convert & status

  • Convert – runs the conversion from the current Input format to the selected Output format.
  • AR2 key display (label / text box) – shows the current AR2 encryption key when AR-V2 mode is in use. It updates from the AR2 key dialog and AR2 seeds.
  • Status / errors – some errors appear as message boxes; detailed per-cheat errors are written into the Output as comment blocks (see section 8).

3. Quick start

3.1 Basic RAW → PNACH example

  1. Choose Input → Unencrypted → Standard(RAW).
  2. Choose Output → Unencrypted → Pnach(RAW).
  3. Paste RAW codes into the Input box, for example: 
    90000000 00000000
  4. Ensure Game Name is filled in appropriately.
  5. Optionally click Options → Set PNACH CRC... to configure CRC and game name for PNACH.
  6. Click Convert.
  7. The Output box now shows PNACH-style lines: 
    // Converted by Omniconvert C#
// CRC: XXXXXXXX, Game: Your Game Name

[Enable Code (Must Be On)]
patch=1,EE,90000000,extended,00000000
  8. Use File → Save As → .pnach to save.

3.2 Converting between consoles/devices

  1. Select the correct Input device/crypt from the Input menu (e.g. ARMAX, CodeBreaker, GS, etc).
  2. Select the desired Output device/crypt in the Output menu.
  3. Paste your codes in the Input box.
  4. Click Convert, then review the Output.
  5. Use File → Save As to export a text file, CBC, ARMAX BIN, P2M, PNACH, or Swap Magic file as needed.

4. File menu: loading & saving

4.1 Loading

File → Load supports:

  • Text File... – loads a plain text file into the Input box.
  • AR MAX .bin... – parses an ARMAX save file and imports its cheats into the Input box.
  • CodeBreaker CBC... – reads a CBC file, extracts cheats, and loads them into the Input box as RAW codes (with labels).
  • XP/GS P2M... – reads an XP/GS file, populates the Game Name box, and loads its cheats as RAW codes.

After loading a binary format (ARMAX, CBC, P2M) the tool converts those codes to RAW internally so you can freely convert them to other formats.

4.2 Saving

File → Save As supports:

  • Text... – saves the Output box content as a .txt file.
  • .pnach... – saves the Output box as a PNACH file (for PCSX2). If the Output is empty you’ll be asked to confirm.
  • AR MAX File (.bin) – creates an ARMAX binary file using the last converted cheats and Game ID.
  • CBC File (.cbc) – writes a CodeBreaker CBC file (you can choose Save Version 7 or 8 under Options).
  • XP/GS File (.P2M) – saves cheats in P2M format for XP / GS.
  • Swap Magic File (.bin) – creates a small Swap Magic Coder file.

ARMAX, CBC, P2M and Swap Magic saving require that a conversion has run successfully (the tool uses the last converted cheats and Game ID). If nothing has been converted yet, those options may show warnings or produce empty data.

5. Input & Output formats

5.1 Unencrypted group

Under Unencrypted you’ll find RAW-style formats:

  • Standard(RAW) – generic address/value pairs like: 
    XXXXXXXX YYYYYYYY
  • INLINE (from ', CRYPT_XXXX' headers) NEW – special input-only mode, see section 6.
  • Pnach(RAW) – PNACH-compatible patch lines (input or output) with patch=1,EE,ADDR,extended,VALUE syntax.
  • ARMAX(RAW) – ARMAX RAW codes (not encrypted text).
  • AR/GS V1-2(RAW) – Action Replay / GameShark v1–2 RAW.
  • Codebreaker(RAW) – CodeBreaker RAW format.
  • XP/GS V3(RAW) – XP / GameShark v3-style RAW format.

5.2 Action Replay group

Under Action Replay you’ll find the AR crypted formats:

  • AR-V1
  • AR-V2 (AR2 – uses the AR2 key and seeds; see Options → Set AR2 Key Code)
  • ARMAX (encrypted ARMAX text codes, e.g. VF78-9CQT-6TRXX)

5.3 CodeBreaker group

  • CodeBreaker – standard CodeBreaker encrypted format.
  • CodeBreaker (Day1 / v7) – variant for v7 Day1 codes.
  • CodeBreaker (v8 / CFU) – variant for v8+ CFU format.

5.4 GameShark & Xploder groups

  • GameShark → InterAct – GS v1 / v2.
  • GameShark → MadCatz – GS v3–4 and GS v5+.
  • Swap Magic → Coder V3.x – simple Swap Magic Coder format.
  • Xploder (XP) – XP v1–3, v4, and v5+ modes.

When you choose a device+crypt from the Input menu, Omniconvert expects codes in that exact format. The Output menu controls how those codes will be re-emitted.

6. INLINE input mode NEW

INLINE is an input-only mode that lets you mix different crypt types in a single paste, as long as each cheat block has a header line with a , CRYPT_XXXX tag.

6.1 What INLINE does

  1. You select Input → Unencrypted → INLINE (from ', CRYPT_XXXX' headers).
  2. You paste text containing multiple cheats, each starting with a label line.
  3. Each label line includes , CRYPT_XXXX, for example , CRYPT_ARMAX.
  4. Omniconvert reads each label, identifies the crypt, and sets that crypt/device for that cheat only.
  5. All cheats are then converted into the single Output format you selected (e.g. PNACH, CodeBreaker, RAW, etc.).

6.2 Supported CRYPT tags

INLINE recognises the following identifiers in headers:

  • CRYPT_AR1
  • CRYPT_AR2
  • CRYPT_ARMAX
  • CRYPT_CB
  • CRYPT_CB7_COMMON
  • CRYPT_GS3
  • CRYPT_GS5
  • CRYPT_MAXRAW
  • CRYPT_RAW

The tag is case-insensitive, but using the exact uppercase form (e.g. CRYPT_ARMAX) is recommended for clarity.

6.3 Required formatting

Each cheat (or cheat block) should look like:

<Label line with , CRYPT_XXXX>
<code line 1>
<code line 2>
...

<Next label line with , CRYPT_YYYY>
<code line 1>
<code line 2>
...

Example: ARMAX + AR2 + CodeBreaker in one paste

(M) by Codejunkies , CRYPT_ARMAX
VF78-9CQT-6TRXX
9Y3Z-DHNW-3QXAJ
UW8R-MMUW-X8P31

(M) Must Be On by InterAct , CRYPT_AR2
0E3C7DF2 1853E59E
EE85F492 BCC65302

Enable Code (Must Be On) by Code Master , CRYPT_CB
FA7A006E 32B24109
  • The first block is treated as ARMAX input.
  • The second block is treated as AR2 input.
  • The third block is treated as CodeBreaker input.

INLINE also supports ARMAX text codes like VF78-9CQT-6TRXX. These are auto-detected and decoded just as they are in plain ARMAX input mode.

6.4 Output behaviour of the , CRYPT_XXXX tag

Non-PNACH outputs

  • The label’s , CRYPT_XXXX is updated to match the output crypt.
  • If there was no CRYPT tag, one is added at the end of the label.

Example (Output = CodeBreaker RAW):

Input label:  (M) by Codejunkies , CRYPT_ARMAX
Output label: (M) by Codejunkies , CRYPT_CB

PNACH (RAW) output

  • When Output is Pnach(RAW), the , CRYPT_XXXX part is removed from section names.

Example PNACH output section:

[Enable Code (Must Be On)]
patch=1,EE,90000000,extended,00000000

6.5 Fallbacks

  • If a label doesn’t contain , CRYPT_XXXX, the cheat uses the current input device/crypt from the menu.
  • If the tag uses an unknown crypt name, it is treated conservatively (RAW) and converted as far as possible.

7. PNACH (RAW) input & output UPDATED

7.1 PNACH as output

When you select Output → Unencrypted → Pnach(RAW), Omniconvert emits PCSX2-friendly patch lines: 

// Converted by Omniconvert C#
// CRC: XXXXXXXX, Game: Your Game Name

[Group\Code Name]
patch=1,EE,ADDR,extended,VALUE
  • Each cheat becomes a PNACH-style section.
  • Group / code names come from your labels or auto-generated names.
  • Section headers never include , CRYPT_XXXX (even in INLINE mode).

7.2 PNACH as input

Selecting Input → Unencrypted → Pnach(RAW) tells Omniconvert to parse patch=1,EE,ADDR,extended,VALUE lines as codes. You can then convert PNACH codes back to RAW or other devices.

7.3 PNACH CRC dialog NEW

Options → Set PNACH CRC... opens a helper dialog that lets you:

  • Type or paste an 8-digit CRC (hex).
  • Type or adjust the Game Name string used in PNACH headers.
  • Pick a known CRC / name from a built-in list (loaded from PnachCRC.json).
  • Browse for a PS2 .ELF or main executable (e.g. SLUS_205.88) to compute the CRC automatically.
  • Drag & drop an ELF file onto the dialog to compute CRC quickly.

The Add CRC checkbox on the main form controls whether this CRC and game name are included in the PNACH output header. Unchecking it will output PNACH patches without CRC / name comments.

8. Per-cheat error reporting NEW

Older builds could silently drop cheats if they failed mid-conversion. In v1.01 this has been changed:

  • Each cheat is converted independently.
  • If a cheat fails, an error comment block is added to the Output instead of silently vanishing.

8.1 Error comment format

For a failing cheat, you’ll see something like:

// [ERROR] Enable Code (Must Be On)
// Error code: 3
// Failed to decrypt AR2 code – key mismatch, invalid seeds, or corrupted line.
  • The first line names the logical cheat (group / code name).
  • The numeric error code is printed for advanced debugging.
  • A friendly description is appended (from Translate.TransGetErrorText()).
  • A blank line follows, then conversion continues with the next cheat.

This applies to both PNACH and non-PNACH outputs. In PNACH mode the error block is written instead of the code’s patch lines; in other outputs it appears where the cheat’s codes would normally be.

9. Options menu: advanced settings

9.1 ARMAX Options...

Options → AR MAX Options... opens a dialog where you can set:

  • Game ID used by ARMAX exports and organizers.
  • Region – USA / PAL / Japan (affects internal ARMAX metadata).
  • Verifiers – Automatic or Manual mode (influences ARMAX disc verification behaviour).
  • Disc Hash Drive – which optical drive letter to use for ARMAX disc-hash calculation, or “Do Not Create”.

These settings sync with the main Game ID box and the ARMAX “Disc Hash” menu.

9.2 ARMAX Disc Hash

Options → AR MAX Disc Hash lets you choose:

  • Do Not Create – disables disc-hash generation.
  • Use Drive X:\ entries – pick which CD/DVD drive to use to read disc data for hash generation.

9.3 AR2 Key Code...

Options → Set AR2 Key Code... opens the AR2 key dialog:

  • Shows the current AR2 encrypted key derived from the seeds.
  • Lets you enter a new 8-hex-digit AR2 key manually.
  • Provides a list of common keys to pick from.
  • Updates the main window’s AR2 key display when you click OK.

9.4 ARMAX Verifiers (Automatic / Manual)

Under Options → AR MAX Verifiers you can choose whether ARMAX disc verification is managed automatically by the tool or left in manual mode.

9.5 ARMAX Region

Options → AR MAX Region toggles the region flag used in ARMAX data:

  • USA
  • PAL
  • Japan

9.6 CBC Save Version

Options → CBC Save Version selects how CBC files are written:

  • Version 7 (Day1) – for older Day1-style CBC files.
  • Version 8 (CFU) – for newer CFU-style CBC files.

9.7 GS3 Key

Options → GS3 Key sets the internal key index for GameShark v3+ conversions. Keys 0–4 are available; which one you need depends on your GS build.

9.8 Make Organizers

Options → Make Organizers controls whether “organizer” entries are generated for ARMAX / CodeBreaker (special codes that structure how cheats appear in device menus). When enabled, extra codes may be added to help devices display groups and folders more cleanly.

9.9 PNACH CRC...

Covered in detail in section 7.3. In short, this dialog centralizes CRC, game name and ELF-based CRC calculation for PNACH output.

10. Tips & notes

  • Always verify on hardware / emulator. Conversion success doesn’t guarantee a cheat is logically correct; test everything.
  • Use INLINE to clean messy mixed lists. If you have big dumps from forums or legacy tools where different crypt formats are intermixed, adding , CRYPT_XXXX tags lets you convert the whole thing in one go.
  • Watch the error blocks. If any cheat fails to convert, an // [ERROR] block explains why – usually misformatted lines, wrong key, or unsupported code type.
  • Game ID vs CRC. Game ID is used for ARMAX / organizers / some save formats; CRC is used for PNACH / PCSX2. They’re separate values and can both be set.
  • Back up codes. Before batch-converting or saving over existing files, keep a copy of your original codes and save files.
