1. Overview

Omniconvert C# is a reimplementation of the classic Omniconvert tool, written in C# / WinForms. It converts PlayStation 2 cheat codes between various devices and formats:

  • Action Replay V1 / V2
  • Action Replay MAX (ARMAX) – encrypted and RAW
  • GameShark (various versions)
  • CodeBreaker (CBC, encrypted & RAW)
  • XP / Xploder
  • Standard RAW
  • PCSX2 PNACH (RAW patch format)
  • Swap Magic Coder

The C# version aims to behave like the original Omniconvert, while adding a few extra conveniences for modern workflows:

  • Inline crypt headers (, CRYPT_XXXX) so you can paste mixed devices at once.
  • Native PNACH input/output, including CRC management and a CRC “Add CRC” toggle.
  • Refinements around ARMAX, CBC, XP/GS and Swap Magic export.

Version 1.01 (C# modular build) adds:

  • INLINE input mode – per-cheat , CRYPT_XXXX tags in one paste.
  • PNACH (RAW) input & output with CRC support and a dedicated CRC dialog.
  • Per-cheat error reporting – failed cheats are preserved with clear error comment blocks instead of silently disappearing.
  • UI tweaks such as AR2 key display and PNACH CRC “Add CRC” checkbox integration.

Version 1.03 adds:

  • Drag & drop loading – you can now drop supported files directly onto the Input box instead of using File → Load. v1.03
  • ARMAX button glyph decoding – controller button icons embedded in ARMAX names and notes are converted into readable tokens like L1, L2, X, Triangle when importing .bin files. v1.03
  • ARMAX notes appended to labels – non-empty game and cheat notes from ARMAX .bin files are appended in braces after the label text, for example Infinite Time {Press L1+L2+X to activate.}. v1.03
  • Smarter PNACH file naming – when Output is Pnach(RAW) and a PNACH CRC has been configured, the default .pnach filename is built from the ELF name and CRC (for example SLUS-20672_12345678.pnach). v1.03

2. Main window tour

2.1 Input panel (left)

  • Input text box – paste or type your source codes here.
  • Clear – clears the input box.
  • Game ID (Input) – numeric ID used by some devices (e.g., ARMAX, organizers).
  • Input format label – shows the current input selection, for example: Input: Standard(RAW) or Input: INLINE (from ', CRYPT_XXXX' headers).

2.2 Output panel (right)

  • Output text box – converted output appears here after pressing Convert.
  • Clear – clears the output box.
  • Game ID (Output) – numeric ID used by some devices for the output format (e.g., ARMAX).
  • Output format label – shows the current output selection, for example: Output: AR2(RAW) or Output: Pnach(RAW).

2.3 Bottom row

  • Convert – runs the conversion from the current Input format to the current Output format.
  • Game ID – a shared Game ID used for organizers / binary formats (ARMAX, CBC, XP/GS, etc.).
  • Game Name – optional description used when creating ARMAX/CBC/P2M/Swap Magic files.

3. Quick start

  1. Paste your source codes into the Input box.
  2. Select the correct Input format from the Input dropdown.
  3. Select your desired Output format from the Output dropdown.
  4. Click Convert.
  5. Copy the results from the Output box, or use File → Save As to save them.

If you see error comments (e.g., # ERROR: ...) in the Output, the converter preserved the original lines but could not convert them. See section 9.

4. File menu: loading & saving

4.1 Loading

File → Load supports:

  • Text File... – loads a plain text file into the Input box.
  • AR MAX .bin... – parses an ARMAX save file and imports its cheats into the Input box.
  • CodeBreaker CBC... – reads a CBC file and loads them into the Input box as RAW codes (with labels).
  • XP/GS P2M... – reads an XP/GS file, populates the Game Name box, and loads its cheats as RAW codes.

After loading a binary format (ARMAX, CBC, P2M) the tool converts those codes to RAW internally so you can freely convert them to other formats.

ARMAX notes and controller buttons v1.03

As of v1.03, when you load an AR MAX .bin file, Omniconvert C# preserves both the game/cheat notes and the special controller button icons stored in those notes:

  • If a game or cheat has a non-empty note/comment, it is appended in curly braces to the imported label text, for example:
    Infinite Time {Press L1+L2+X to activate.}
  • Internal AR MAX button glyph codes are decoded to readable tokens in both names and notes. For example:
    • 0x0010 → X
    • 0x0011 → Square
    • 0x0012 → Triangle
    • 0x0013 → Circle
    • 0x001A → L1, 0x001B → L2
    • 0x001C → R1, 0x001D → R2
    • 0x0016 → Up, 0x0017 → Right, 0x0018 → Down, 0x0019 → Left

4.2 Saving

File → Save As supports:

  • Text... – saves the Output box content as a .txt file.
  • .pnach... – saves the Output box as a PNACH file (for PCSX2). If the Output is empty you’ll be asked to confirm. As of v1.03, when Output is Pnach(RAW) and a PNACH CRC has been configured via Options → Set PNACH CRC..., the default filename is built from the ELF name and CRC (for example SLUS-20672_12345678.pnach).
  • AR MAX File (.bin) – creates an ARMAX binary file using the last converted cheats and Game ID.
  • CBC File (.cbc) – writes a CodeBreaker CBC file (you can choose Save Version 7 or 8 under Options).
  • XP/GS File (.P2M) – saves cheats in P2M format for XP / GS.
  • Swap Magic File (.bin) – creates a small Swap Magic Coder file.

ARMAX, CBC, P2M and Swap Magic saving require that a conversion has run successfully (the tool uses the last converted cheats and Game ID). If nothing has been converted yet, those options may show warnings or produce empty data.

4.3 Drag & drop loading v1.03

As of v1.03, you can also load files by dragging them from Explorer and dropping them onto the Input box. Supported file types are:

  • .txt – same as File → Load → Text File...
  • .cbc – same as File → Load → CodeBreaker CBC...
  • .bin – same as File → Load → AR MAX .bin...
  • .p2m – same as File → Load → XP/GS P2M...

The drag & drop loader uses the same parsing and cleaning logic as the File → Load commands; it is simply a faster way to get supported files into the Input window.

5. Input & Output formats

5.1 Unencrypted group

Under Unencrypted you’ll find RAW-style formats:

  • Standard(RAW) – generic address/value pairs like: 
    XXXXXXXX YYYYYYYY
  • INLINE (from ', CRYPT_XXXX' headers) NEW – special input-only mode, see section 6.
  • Pnach(RAW) – PNACH-compatible patch lines (input or output) with address / data pairs.

5.2 Encrypted group

The Encrypted group lists device-specific encrypted formats:

  • AR1(RAW) / AR2(RAW) – Action Replay V1 / V2 RAW-like forms.
  • GS(RAW) – GameShark RAW-like forms.
  • ARMAX – encrypted ARMAX codes using the current ARMAX options.
  • CodeBreaker – encrypted CBC codes (Day1/CFU style).
  • XP/GS – XP / Xploder style codes.

When you choose an encrypted format as Output, Omniconvert C# internally converts from whatever input device you selected to a common RAW form, then re-encrypts using the appropriate keys.

6. INLINE input mode

INLINE mode lets you paste codes with per-cheat crypt headers such as: 

+My ARMAX code , CRYPT_ARMAX
F0123456 12345678
00000000 00000000

+My RAW code , CRYPT_STD
20123456 00000001

When Input is set to INLINE (from ', CRYPT_XXXX' headers), the tool does the following:

  • Scans for lines ending in , CRYPT_XXXX (case-insensitive).
  • Treats the header as the code’s name/label, and the following lines as code lines until a blank line.
  • Uses the specified CRYPT_XXXX as the input device/crypt for just that block.

This is useful when converting mixed device lists, for example copying from a forum post that contains ARMAX, CodeBreaker and RAW in one big paste.

7. PNACH CRC dialog & Add CRC

When using PNACH formats you’ll see a small Add CRC checkbox near the Output label and a corresponding option under Options → Set PNACH CRC....

  • Set PNACH CRC... – lets you enter or auto-detect the game CRC and ELF name.
  • Add CRC – when checked, PNACH output includes the CRC comment header.

The C# version keeps PNACH-specific state (CRC, ELF name, Add CRC toggle) separate from the core device/crypt logic, so standard device conversions are not affected.

8. ARMAX options window

The ARMAX options window centralizes a few ARMAX-related controls that were previously scattered across the menus:

  • Region – choose USA / PAL / Japan region for ARMAX codes.
  • Verifiers – automatic or manual verifier selection.
  • Disc hash – whether to generate ARMAX disc-hash entries.

You can open it from Options → ARMAX options... or via the related toolbar/menu shortcuts.

9. Errors & per-cheat comments

Instead of silently discarding invalid codes, Omniconvert C# preserves them as commented blocks with a short explanation. For example: 

# ERROR: Unsupported GS code type 0xFF for this device
# 12345678 00000000

This makes it easier to identify which cheats failed and why, without losing the original data.

You can keep or delete these blocks manually. They are just comments in the output text.

10. Tips & notes

  • Always double-check the Input format before converting.
  • If a device has multiple key variants (e.g., GS3 keys) ensure the correct key is selected under Options.
  • Use INLINE mode when working with forum posts that mix several crypts.
  • When in doubt, convert to Standard(RAW) first, then to the final target device.
