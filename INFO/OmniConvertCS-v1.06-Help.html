1. Overview

OmniConvertCS is a native C++17 continuation of the classic OmniConvert 1.1.1R PlayStation 2 cheat-code converter. The application keeps the OmniConvertCS name for continuity with earlier releases.

Version 1.06.02 replaces the former C# / WinForms application with a native 64-bit Windows desktop build while retaining the familiar conversion workflow.

Native C++17
No .NET runtime required.
Windows x64
Windows 7 SP1 and newer.
Mixed Crypts
Per-code INLINE crypt selection.
PS2 MIPS
R5900 assembler and disassembler.
CMP Formatting
Code Database-compatible text output.
PNACH Tools
Input, output, CRC, and filenames.

Supported areas

  • Action Replay V1 and V2
  • Action Replay MAX encrypted and RAW formats
  • CodeBreaker encrypted and RAW formats
  • GameShark and Xploder formats
  • Standard RAW and device-specific RAW variants
  • PCSX2 PNACH input and output
  • INLINE mixed-crypt input
  • PS2 R5900 / MIPS assembly and disassembly
  • Supported ARMAX, CBC, P2M, and related codelist workflows
The previous C# implementation is preserved under ~Legacy~/ for reference. It is not used by the active C++17 build.

2. Main window tour

2.1 Input side

  • Input format – selects the source crypt or text format.
  • Input text box – paste, type, or load the source codes.
  • Clear – clears the input text.
  • Game ID – used by formats that require a numeric game identifier.

2.2 Output side

  • Output format – selects the destination crypt or text format.
  • Output text box – displays the converted result.
  • Clear – clears the output text.
  • Game ID – used by supported output formats that require it.

2.3 Bottom controls

  • Convert – performs the current input-to-output conversion.
  • Swap – exchanges the selected input and output formats where supported.
  • Game Name – optional title used by binary codelists and PNACH helpers.
The window is resizable. Input, output, labels, buttons, and the Game Name field are repositioned and repainted automatically.

3. Quick start

  1. Select the correct Input format.
  2. Select the desired Output format.
  3. Paste or load the source codes into the Input box.
  4. Enter a Game ID or Game Name when required by the selected format.
  5. Click Convert.
  6. Copy the result or save it from the File menu.
Always verify the selected input crypt before converting. A valid-looking result can still be incorrect when the wrong source crypt is selected.

4. Loading and saving

4.1 Loading

The File menu and drag-and-drop support may load formats such as:

  • Plain text files
  • PNACH files
  • ARMAX .bin codelists
  • CodeBreaker CBC files
  • XP / GameShark P2M files
  • PS2 ELF files for PNACH CRC calculation

Supported files can also be dropped onto the Input area. The same parser and cleanup logic is used whether the file is loaded from the menu or by drag-and-drop.

ARMAX names, notes, and controller glyphs

  • Supported ARMAX button glyphs are converted to readable names.
  • Available game and cheat notes are preserved where possible.
  • Names, folders, and descriptions are retained when the destination format supports them.

4.2 Saving

Depending on the selected output format, Save As may create:

  • Plain text files
  • PCSX2 .pnach files
  • ARMAX .bin files
  • CodeBreaker CBC files
  • XP / GameShark P2M files
  • Other supported format-specific files
Binary exports use the most recent successful conversion. Run Convert before saving a binary codelist and supply any required Game ID or Game Name.

5. Input and output formats

5.1 RAW-style formats

RAW codes normally use two hexadecimal words:

XXXXXXXX YYYYYYYY

Available RAW families may include:

  • Standard RAW
  • Action Replay / GameShark V1–V2 RAW
  • CodeBreaker RAW
  • GameShark / Xploder V3 RAW
  • ARMAX RAW

5.2 Encrypted formats

Encrypted choices may include:

  • Action Replay V1
  • Action Replay V2
  • Action Replay MAX
  • CodeBreaker V1–V6
  • CodeBreaker V7+ Common
  • GameShark V3–V4 / Xploder V4
  • GameShark V5+ / Xploder V5+

5.3 Special text formats

  • INLINE – mixed source crypts in one input.
  • PNACH – PCSX2 patch text.
  • PS2 MIPS – R5900 assembly text and opcodes.

6. INLINE mixed-crypt input

INLINE mode lets each named code block declare its own source crypt. This is useful for lists containing several devices or encryption types. 

Infinite Health, CRYPT_AR2
CEB1E0D2 BCA99C84

Master Code, CRYPT_ARMAX
VNVH-7KMA-DX61K
A4UT-3BKQ-TYYV5

Max Money, CRYPT_CRAW
20123456 05F5E0FF

Recognized INLINE identifiers

CRYPT_AR1
CRYPT_AR2
CRYPT_ARMAX
CRYPT_CB
CRYPT_CB7_COMMON
CRYPT_GS3
CRYPT_GS5
CRYPT_MAXRAW
CRYPT_RAW
CRYPT_ARAW
CRYPT_CRAW
CRYPT_GRAW
  • Headers are case-insensitive.
  • Each named block is processed with its own crypt.
  • A block without a recognized header defaults to RAW.
  • INLINE is an input-only format.

7. PNACH and CRC helper

7.1 PNACH input

PNACH input can contain group names and EE patch rows:

[Infinite Health]
patch=1,EE,20123456,extended,00000063

Metadata such as gametitle=, comment=, crc=, and description= is handled separately from code rows.

7.2 PNACH output

PNACH output uses PCSX2-compatible patch rows:

patch=1,EE,20123456,extended,00000063

7.3 PNACH CRC helper

The CRC helper calculates the PCSX2 game CRC from a PS2 ELF and associates it with an ELF name and game title.

Mappings are stored in:

PnachCRC.json

Example mapping:

SLES_526.41>A1FD63D6>Game Name
  1. Select PNACH as the output format.
  2. Open Set PNACH CRC or enable Add CRC.
  3. Drop or browse to the PS2 ELF.
  4. Confirm the detected CRC and Game Name.
  5. Save the mapping.

A mapped PNACH filename may be suggested as:

SLUS_203.12_A1FD63D6.pnach

8.1 Transpose Type

  • Original – preserves the original OmniConvert transpose behavior and is the default for new settings.
  • Strict – uses stricter transpose handling.

8.2 CMP Output Mode

CMP Output Mode formats supported text output for the CMP Code Database.

Example source:

Master Codes
Enable Code (Must Be On), by Lajos Szalay
904903A8 0C109375
2012EF74 00441021

CMP-formatted output:

!Master Codes:
+Enable Code (Must Be On)
%Credits: Lajos Szalay
$904903A8 0C109375
$2012EF74 00441021
!!
  • A name-only section opens a group with !Group Name:.
  • !! closes the group before it.
  • The previous group is closed before another group begins.
  • An open group is closed when the input ends.
  • Inline , by Author text becomes a %Credits: line.
  • CMP formatting is not applied to PNACH or PS2 MIPS output.

8.3 MGS C-Type Pointer Mode

MGS C-Type Pointer Mode is for Metal Gear Solid CodeBreaker lists that use a custom engine patch where type C acts like a pointer write instead of the normal CodeBreaker C-type gate. 

CAAAAAAA OOOOVVVV

When enabled for CodeBreaker-family input, the line is normalized after decryption into:

6AAAAAAA 0000VVVV
00010001 0000OOOO

After normalization, the normal output selection controls the final result. For example, GameShark output will receive the GameShark pointer-style equivalent instead of a GameShark C code.

9. ARMAX options

The ARMAX options control settings such as:

  • USA, PAL, or Japan region
  • Automatic or manual verifier selection
  • Disc-hash generation
  • Other ARMAX-specific output behavior

Open the dialog from the Options menu when ARMAX-specific configuration is required.

10. PS2 R5900 / MIPS mode v1.06

OmniConvertCS includes a bidirectional PS2 R5900 assembler and disassembler. It can convert instruction opcodes to readable assembly and assemble supported text back into 32-bit instruction words.

Supported instruction areas

  • Standard R5900 integer operations
  • Branches, jumps, traps, and labels
  • COP0 system-control operations
  • COP1 floating-point operations
  • R5900 MMI0–MMI3 packed multimedia instructions
  • COP2 register transfers and branches
  • Core VU0 macro-mode arithmetic
  • VU destination masks and broadcast forms

Example:

vadd.xyzw $vf1, $vf2, $vf3
vaddz.xw  $vf4, $vf5, $vf6
vmul.y    $vf7, $vf8, $vf9
Reserved, invalid, or unsupported encodings are preserved as exact .word values whenever decoding them as an instruction could alter the original data.

11. Errors and preserved data

Invalid or unsupported input should not silently disappear. Depending on the selected format, OmniConvertCS preserves the original material and reports a conversion error or emits an exact RAW/.word representation.

Example text error block:

# ERROR: Unsupported code type for the selected format
# 12345678 00000000

Review any error block before using the generated codes.

12. Command-line interface

OmniConvertCS can also be run from CMD or PowerShell for scripted and batch conversions.

12.1 General usage

OmniConvertCS.exe [-a|-af] [-m|-mc] [-e] [-o <outdir>] <Output-Crypt> <input.txt | input_folder>
OmniConvertCS.exe -h

12.2 Common options

  • -a – analyze supported RAW-family output.
  • -af – analyze and try alternate input crypts when supported.
  • -m – enable manual choices during supported auto-fix workflows.
  • -mc – review every cheat manually.
  • -e – apply the supported CRAW E001-to-D cleanup helper.
  • -o <dir> – select the output root.
  • -h or --help – display CLI help.

12.3 Manual-review commands

  • Enter – continue the normal attempt order.
  • S – skip the current cheat.
  • D – delete the current cheat from output.
  • Q – stop processing.
  • N -y – force-accept crypt number N for that attempt.
  • N -kHEX – override an AR2 key for that attempt.

12.4 CRAW E001 cleanup

For supported CRAW output, -e can rewrite:

E001VVVV 0AAAAAAA

as:

DAAAAAAA 0000VVVV

14. Tips and notes

  • Confirm the source crypt before every conversion.
  • Use Standard RAW as an intermediate format when troubleshooting.
  • Use INLINE mode for lists containing several crypt formats.
  • Enable CMP Output Mode only when preparing compatible database text.
  • Keep CMP mode disabled for ordinary forum or device output.
  • Supply the correct ARMAX region and verifier settings.
  • Review MIPS .word output before assuming it is an executable instruction.
  • Keep a backup of important original code lists before batch processing.
